### D-011. B2C 우선 타겟
**상태:** Accepted
**시점:** 2026-05-02
**배경:**
ARCC가 B2C(개인) / B2B(기업 직원) / B2G(군/공공) / B2H(병원/재활) 4갈래 사업 모델 가능. 한정된 자원으로 동시 추진 시 모든 채널이 미완성될 위험.
**결정:**
B2C 개인 구독을 1차 타겟으로 우선 집중. B2B/B2G/B2H는 B2C 검증 후 단계적 확장.
**근거:**

B2C는 빠른 피드백 루프 (사용자 반응 즉시 확인)
누적된 개인 코칭 히스토리 = ARCC의 차별점이자 락인 효과
본인(50대 직장인 + 하프마라톤 준비)이 1번 타겟 사용자 → dogfooding으로 품질 검증
B2H는 의료법/개인정보보호 이슈로 변호사 자문 필수 → 후순위

**검증:**
2026-09~10 정식 런칭 후 B2C 사용자 100명 도달 시점에 B2B 진출 검토.

### D-013. 인프라 단계적 확장 ($45 → $130/월)

**시점:** 2026-05-02
**배경:**

**결정:**
2단계 인프라 로드맵 채택.

정식 런칭 (2026-09~10): Render.com + Supabase Pro + 도메인 ≈ $130/월

**근거:**

Supabase Free는 7일 비활성 시 일시정지 → 런칭 전 Pro 업그레이드 필수

## D-032: 메모2 (다음 러닝 메모) 폐기

**Status**: Accepted  
**Date**: 2026-05-09 (KST)  
**Related**: D-031 (변경 4 폐기), Q-003 (정의 재정렬), Q-014/Q-015/Q-016 (정리 필요)

### Context

5/9 논의에서 메모2(다음 러닝 메모) 도입 검토. 제이슨이 5/9 메시지로 다음 기능을 제안:
- 메모1 (러닝 데이터 업로드 시): D-031 변경 2로 이미 박제
- 메모2 (분석 이후, 다음 러닝에 대한 메모): 신규 검토 대상
  - 예: "내일 휴식" → ARCC AI가 "내일 휴식, 모레 이지런 50분 권장" 응답

쟁점 1~3 결정 진행:
- 쟁점 1 (메모2 위치): 후보 A 채택 (분석 결과 화면 하단, 컴포넌트 분리 설계)
- 쟁점 2 (입력 시 AI 응답): 시나리오 2 채택 (저장만, 다음 분석 시 반영)
- 쟁점 3 (Q-003 관계): 해석 C 채택 (메모2 = Q-003 입력 채널)

쟁점 4 (1단계 범위) 단계에서 제이슨이 본질적 의문 제기:
> "메모2이 필요없을 수 있겠는데. 내일 휴식을 미리 알림할 필요가 없을듯. 5/8 분석(내일은 이지런 추천), 5/9 휴식, 5/10 러닝데이터 분석 시 5/9 데이터 없음 확인하고 5/10 러닝데이터와 5/9 휴식 적용하여 분석하겠지"

**즉, 데이터 부재 = 휴식 사실**을 Claude가 자동 추론 가능. 사용자가 메모2로 입력할 필요 없음.

### Decision

1. 메모2 (다음 러닝 메모) 기능 도입하지 않음
2. 1단계 메모 기능 = **메모1만** (D-031 변경 2 그대로 유지)
3. 다음 러닝 추천 (Q-003)은 **데이터 기반 자동 추론**으로 정의

### Rationale

1. **데이터 부재 = 휴식 추론 가능**: 사용자 입력 불필요
2. **부상/특이사항 메모1로 커버**: 러닝 직후 메모1에 "발목 회복 중" 등 기록 → 다음 분석 시 반영

def _calc_fi(splits, summary):
    if len(splits) < 4: return 30
    # 페이스 초 변환
    if len(paces) < 4: return 30
    half = len(paces) // 2
    first_half_avg = sum(paces[:half]) / half
    second_half_avg = sum(paces[half:]) / (len(paces) - half)
    decline_pct = ((second_half_avg - first_half_avg) / first_half_avg) * 100
    # HR drift
    hr_drift = (sum(hrs_second)/len(hrs_second)) - (sum(hrs_first)/len(hrs_first))
    fi = max(0, min(100, round(decline_pct * 5 + hr_drift * 1.5)))
    return fi
```

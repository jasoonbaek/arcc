## [버그1] 모바일 CSV 업로드 안 됨 (우선순위: 🔴 높음)

### 현상
- **환경**: 모바일 엣지 브라우저, 모바일 크롬
- **화면**: 러닝 기록 업로드
- **증상**: CSV 파일이 탐색기에서 선택되지만, ARCC 앱에 로딩되지 않음
- **발생일**: 2026년 4월 12일

### 원인 추정
파일 input의 `accept` 속성이 모바일 브라우저에서 제대로 인식되지 않음

### 수정 방법

```html
<!-- 변경 전 (현재 추정) -->
<input type="file" accept=".csv" />

<!-- 변경 후 -->
<input 
  type="file" 
  accept=".csv, text/csv, text/plain, application/vnd.ms-excel, .txt"
  id="csv-upload"
/>
```

### 추가 수정
- 파일 선택 후 **로딩 인디케이터** 표시
- 5초 이상 로딩 시 "파일 처리 중..." 메시지 표시
- 실패 시 구체적인 오류 메시지 표시 (단순 "업로드 실패" 금지)

## [버그2] 마우스 스크롤 시 화면 이동 (우선순위: 🔴 높음)

### 현상
- **환경**: 노트북 (PC)
- **화면**: 모든 화면 추정
- **증상**: 마우스 스크롤 동작 시 의도치 않게 화면 전체가 이동
- **발생일**: 2026년 4월 12일

### 원인 추정
가로 스크롤 또는 터치 제스처가 활성화되어 있거나, 스와이프 네비게이션이 작동

### 수정 방법

```css
/* 가로 스크롤 방지 */
html, body {
  overflow-x: hidden;
  overscroll-behavior-x: none;
}

/* 스와이프 네비게이션 방지 (필요시) */
body {
  overscroll-behavior: none;
}
```

```javascript
// 가로 스크롤 이벤트 차단 (필요시)
document.addEventListener('wheel', function(e) {
  if (Math.abs(e.deltaX) > Math.abs(e.deltaY)) {
    e.preventDefault();
  }
}, { passive: false });
```

## [버그3] 로그인/회원가입 입력값 초기화 안 됨 (우선순위: 🔴 높음)

### 현상
- **환경**: 노트북 (PC)
- **화면**: 로그인 화면, 회원가입 화면
- **증상**: 로그인 ↔ 회원가입 화면 전환 시 이메일, 비밀번호 입력값이 남아있음 (새로고침 안 됨)
- **발생일**: 2026년 4월 12일

### 원인 추정
SPA 방식에서 화면 전환 시 폼 상태가 초기화되지 않음

### 수정 방법

```javascript
// 화면 전환 시 폼 초기화
function switchToLogin() {
  document.getElementById('signup-form')?.reset();
  // 또는 각 입력 필드 개별 초기화
  document.getElementById('email-input').value = '';
  document.getElementById('password-input').value = '';
  // 로그인 화면 표시
}

function switchToSignup() {
  document.getElementById('login-form')?.reset();
  document.getElementById('email-input').value = '';
  document.getElementById('password-input').value = '';
  // 회원가입 화면 표시
}
```

### 추가 고려
- 브라우저 자동완성 방지가 필요한지 확인
- `autocomplete="off"` 속성 추가 여부 검토
